Welcome to SMSLib for Java.

What is SMSLib?

SMSLib for Java is a Java API which you can use in order to send and/or receive SMS Messages through a GSM modem or phone connected to your PC.

Installation Checklist.

SMSLib is just a jar file that you must copy in your classpath or in Java's "lib/ext" directory.
You should also install the following software components:

  • SUN JDK v1.4.2 or newer.
  • Apache Ant. Used for building the SMSLib sources.
  • Apache Log4J. Used for SMSLib logging needs.
  • A Serial Communications Library. SMSLib can be used either with RxTx or SUN Java Comm v2.0 for Win32
If you are using virtual comm ports on Windows (i.e. ports that are mapped through USB/IrDA/Bluetooth connections) it is advisable to use the Java Comm package.

Installation of SerialComm Libraries.

If you are using SUN Java Comm, do the following copies:

  • Copy "comm.jar" to "(JDKDIR)\jre\lib\ext" directory
  • Copy "javax.comm.properties" to "(JDKDIR)\jre\lib" directory
  • Copy "win32com.dll" file to "(JDKDIR)\jre\bin" directory
If you have a separate JRE, do the same copies in the relevant JRE directories.

If you are using RxTx, do the following copies:

  • Copy "RXTXComm.jar" to "(JDKDIR)\jre\lib\ext" directory
  • Copy "rxtxSerial.dll" file to "(JDKDIR)\jre\bin" directory
  • For Linux, copy "librxtxSerial.so" file to "(JDKDIR)\jre\lib" directory
If you have a separate JRE, do the same copies in the relevant JRE directories.
Be sure to use the correct RxTx binaries (libraries - .dll / .so) for your operating system!

Before first use of SMSLib.

There is a modification that you should do before attemping to use SMSLib. This has to do with the fact that there are two SerialComm Libraries that can be used. Depending on whether you are using RxTx or JavaComm you must alter some imports in the code.

  • CSerialDriver.java: If you are using RxTx, "import gnu.io.*;" should be added. If you are using JavaComm, "import javax.comm.*;" should be added.

Compilation.

SMSLib can be built with the Apache Ant utility. The parameters accepted from the supplied ant script are:

  • clean: Cleans up directories.
  • all: Compiles everything, creates JavaDocs and jar files.
  • compile: Compiles source files only.
  • docs: Creates Javadoc pages.
Before attemping to compile, you should create a xxxx-build.properties file, where xxxx is your machine user name, in which you should set some common paths. Have a look at the tdelenik-build.properties supplied and create a similar one with your own name.

Logging.

SMSLib uses Log4J for its logging needs. By default, its error level is set to WARN and the output is directed to a basic console appender. If you wish to configure Log4J youself with extra settings, you should add a file named "smslib-log.conf" in your apps directory, which will contain the Log4J settings. This file, if available, is read during SMSLib startup and configures its logging.
For your convinience, a sample "smslib-log.conf" is provided in the distribution package, directory "build". This Log4J configuration file enables full debbuging, console-output mode.
Starting from SMSLib v2.1.1, there is a quicker way to enable full, console-output debugging. Just start your application with the "smslib.debug=true" property (the actual value of the property does not matter). When SMSLib sees this property defined, it will set its logger to console and the logging level to all. This is a quick way to enable full debbugging info, without messing with recompilations or log4j definition files.

Example Files.

There are some example files that you can review to see how to implement SMSLib functionality in your own applications.

  • SendMessage.java: Basic logic on how to send a message via your GSM modem/phone
  • SendWapSIMessage.java.java: Basic logic on how to send a WAP PUSH (SI) message.
  • ReadMessages.java: Basic logic on how to read incoming messages from your GSM modem/phone (Synchronous method).
  • ReadMessagesAsync.java: Basic logic on how to read incoming messages from your GSM modem/phone (Asynchronous method).
